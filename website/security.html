Security architecture

16 layers of defense

Nexus OS assumes powerful agents can fail, drift, or be attacked. The answer is not trust. It is layered verification, policy enforcement, and recovery built into the runtime itself.

Defense in depth

Not one wall. Sixteen overlapping systems that catch drift, abuse, bad prompts, broken plans, and risky execution before damage spreads.

Prevent

Block dangerous behavior at the manifest, policy, and sandbox layers.

Observe

Everything important lands in the audit chain and operator surfaces.

Recover

Rollback, cooldowns, and kill gates keep failure bounded.

Security layers

Every action crosses all of these.

The exact layers are visible and explainable. That matters because operators should be able to reason about why an action was allowed, blocked, or rolled back.

1. WASM Dual-Metered Sandbox

Fuel plus epoch interruption prevent infinite loops and resource blowups inside the execution lane.

2. Ed25519 Manifest Signing

Every agent manifest is cryptographically signed so tampering is detectable before an agent gets to act.

3. SHA-256 Audit Chain

Events are hash-linked into an immutable chain; modification breaks continuity and reveals itself.

4. Prompt Firewall

20 injection patterns catch classic role override and extraction attacks before they reach the decision layer.

5. Egress Governor

Agents can only talk to allowed domains and destinations defined for their role.

6. Path Traversal Prevention

Declared paths, path normalization, and sandboxed access stop directory escape behavior.

7. Command Allowlist / Blocklist

Shell access stays governed through explicit command policy and dangerous-command denial.

8. Speculative Execution

Plans can be simulated before reality, which catches destructive mistakes earlier.

9. HITL Consent

High-risk actions route through humans with risk previews instead of silent execution.

10. Fuel Metering

Every action costs fuel, which turns runaway autonomy into a bounded, measurable process.

11. Autonomy Guards

Capability-based access control ensures the agent only touches what its signed role allows.

12. L6 Cool-down

Transcendent agents are forced to pause every 100 cycles so cognition-level drift stays bounded.

13. Time Machine Rollback

System-wide rollback means autonomy can be reversed instead of merely regretted.

14. DID Identity

Each agent receives an Ed25519 identity so actions remain attributable and auditable.

15. JWT Authentication

API and WebSocket surfaces require authenticated access rather than ambient trust.

16. Kill Gates

Emergency circuit breakers can stop one agent, one level, or the whole system immediately.

AUDIT

Immutable evidence

Audit trails, screenshot trails, approval history, and lineage records let you answer the hard operational question: what actually happened?

GOV

Governed action surface

Capabilities, manifests, and allowlists constrain what agents are even allowed to attempt before execution begins.

SIM

Preview before impact

Speculative execution and world modeling reduce surprise by testing risky plans before they touch real state.

UNDO

Recovery as a feature

Rollback, cooldowns, and kill gates make the system survivable when strong autonomy meets a messy world.

Don't trust. Verify.

That is the architecture rule. Agents are useful because the system is allowed to be skeptical of them all the way down.
