# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.x     | Yes       |

## Reporting a Vulnerability

If you discover a security vulnerability, report it privately and responsibly.

1. Do not open a public issue for the vulnerability.
2. Send a detailed report to the project maintainers with:
   - Impacted component(s)
   - Reproduction steps
   - Proof-of-concept or logs (if available)
   - Suggested mitigations (optional)
3. Allow maintainers a reasonable remediation window before public disclosure.

## Responsible Disclosure Process

1. Acknowledge receipt within 3 business days.
2. Triage and severity assessment.
3. Coordinate fix and release timeline.
4. Publish advisory after patch release.
